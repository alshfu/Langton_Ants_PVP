# docs/

Спецификации и техническая документация.

## Файлы

| Файл | Что |
|---|---|
| `interface-contract.md`     | Контракт фронт-бэк: типы AppState, actions, events, форматирование |
| `backend-architecture.md`   | Полная архитектура бэкенда: стек, протокол, БД, инфра, дорожная карта |
| `api/openapi.yaml`          | OpenAPI 3.0 спецификация REST API (генерируется из Fastify + ручные правки) |

## Когда обновлять

- Любое изменение типов в `core/src/contract/` → обновить `interface-contract.md`
- Любое архитектурное решение → апдейт `backend-architecture.md`
- Любое изменение HTTP API → перегенерировать `api/openapi.yaml`

Не редактируем спецификации после реализации (drift). Сначала меняем документ →
обсуждаем → потом код.
